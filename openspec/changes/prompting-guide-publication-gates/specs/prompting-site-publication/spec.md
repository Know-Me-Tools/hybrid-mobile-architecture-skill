## ADDED Requirements

### Requirement: Complete canonical Docusaurus publication
The existing prompting docs-plugin instance SHALL publish the canonical
`docs/prompting/` taxonomy, including start/contract, model routing, six harnesses,
loop guides, ten scenarios, native-agent case study, skill-versus-agent guidance, and
verification/recovery policy.

#### Scenario: Production site is built
- **WHEN** the site production build completes from a fresh install
- **THEN** every required canonical page SHALL have a stable public route and the site
  SHALL contain no hand-maintained duplicate prompting source

### Requirement: Navigation, sitemap, and local search completeness
Autogenerated nested sidebars, sitemap, and deterministic local search SHALL expose all
required prompting pages and preserve clear ownership separate from architecture,
reference-application, and deployment documentation.

#### Scenario: Required page is omitted
- **WHEN** a canonical guide page lacks a public route, sidebar placement, sitemap
  entry, or local-search record
- **THEN** the publication gate SHALL fail with its stable content identifier

### Requirement: KnowMe visual and accessibility contract
Prompting routes SHALL retain KnowMe branding, Flat 2.0 light and dark themes, no
visible borders, separator lines, decorative outlines, gradients, or layout shadows,
and WCAG 2.2 AA navigation, focus, code, diagram, search, and prompt-copy behavior.

#### Scenario: Representative route is viewed in both themes
- **WHEN** a desktop or mobile viewport renders a prompting route
- **THEN** regions SHALL be differentiated by background and spacing, content SHALL
  retain required contrast, and keyboard focus SHALL remain visible without decorative
  layout borders

### Requirement: Static publication boundary
GitHub Pages and the documentation container SHALL publish static documentation only.
No raw private content, authentication service, database, secret, or server-side SaaS
behavior SHALL be embedded in the Pages artifact.

#### Scenario: Site artifact is inspected
- **WHEN** the production artifact is scanned before upload
- **THEN** it SHALL contain only reviewed public assets and SHALL pass the prohibited
  content policy

### Requirement: Existing publication stack is retained
The implementation SHALL retain pinned Docusaurus 3.10.1, the existing local-search
plugin, the branded home/navigation, and the pinned custom GitHub Pages workflow unless
a separately approved change updates them.

#### Scenario: Prompting guide is integrated
- **WHEN** new routes and gates are added
- **THEN** existing architecture, reference-app, deployment, and home routes SHALL
  continue to build and render without framework migration
