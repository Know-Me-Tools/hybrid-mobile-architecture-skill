# ContentBlock UI Contract

`ContentBlock` is the single cross-platform UI contract between `gen_ui_core` (Rust) and
every UI surface. Each A2UI event maps to exactly one variant. The Dart and TypeScript
compilers enforce exhaustiveness at every switch/match — a missing case is a compile error
by design. **Never** render a ContentBlock from a hand-rolled `if`/`else` chain; always
switch over the full discriminated union.

## The 11 variants

| Variant | Purpose | Streaming? |
|---|---|---|
| `text` | Assistant/user prose (Markdown) | yes (`isStreaming`) |
| `thinking` | Extended-reasoning trace, collapsible | yes |
| `code` | Fenced code with `language` + optional `filename` | no |
| `citation` | Text + source list | no |
| `memory` | Recalled/stored memory chip (`memoryType`, `key`) | no |
| `toolUse` | Tool invocation (`toolName`, `input`, `status`) | status transitions |
| `toolResult` | Tool output paired to a `toolUseId` | no |
| `skill` | Skill activation (`skillName`, `status`, `outputSummary`) | status transitions |
| `artifact` | Rendered artifact (document, chart, HTML) | no |
| `image` | Image (`url`/bytes, `alt`, explicit `width`/`height`) | no |
| `divider` | Section separator | no |

The canonical Rust definition lives in `gen_ui_types` (or `gen_ui_core/src/protocol/a2ui.rs`).
The Dart mirror is generated by `flutter_rust_bridge`; the TypeScript mirror lives in
`src/bridge/a2ui/types.ts`. **Never edit a mirror by hand** — regenerate it.

## Rendering rules (both surfaces)

1. **Exhaustive switch, no default fallthrough.** In Dart use a `switch` expression over
   the sealed `ContentBlock` type; in TypeScript use a discriminated `switch (block.type)`
   with a `never`-typed exhaustiveness guard in the `default` branch:
   ```ts
   default: { const _exhaustive: never = block; return _exhaustive }
   ```
2. **One widget/component per variant.** Keep `ContentBlockText`, `ContentBlockCode`, …
   in `gen_ui_widgets` (Flutter) / `@prometheus-ags/gen-ui-react` (React). The dispatcher
   only switches and delegates — no rendering logic inline.
3. **Streaming variants must handle the partial state.** `text`/`thinking` render
   incrementally while `isStreaming` is true; show a caret/shimmer, never a layout jump.
   Animate only compositor-friendly properties (opacity/transform).
4. **`toolUse`/`skill` render status, not raw JSON.** Map `status` (`pending` → `running`
   → `complete`/`failed`) to a designed state with hover/focus affordances. Collapse
   `input`/`outputSummary` behind a disclosure; never dump `Record<string, unknown>`.
5. **`image` requires explicit `width`/`height`** to prevent CLS. `alt` is mandatory (see
   [[a11y-gate]]).
6. **State flows through the layer contract.** Flutter: blocks mutate only via
   `ChatNotifier.streamBlock()` — never direct state assignment. React: block state lives
   in a Zustand store; components receive it through a `useChat`-style hook, never call
   `invoke()`/`listen()` themselves.

## Adding a NEW ContentBlock variant

This changes the frozen union and touches Rust + Dart + TypeScript. Follow **all 7 steps**
in `references/rust/new-block-type.md` — skipping a step produces compile errors by design.
Do not stub a `default` case to "make it compile"; that silently drops the new variant.

After adding a variant, re-run codegen on both surfaces:
- Flutter: `flutter_rust_bridge_codegen generate …` then `flutter pub run build_runner build`
- React: regenerate `src/bridge/a2ui/types.ts` from the Rust source

Then verify a golden/screenshot for the new variant on each surface — see
[[flutter-golden-ui]] and [[tauri-ui-review]].
