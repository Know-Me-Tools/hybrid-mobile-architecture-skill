# Branded documentation portal

Use this recipe to create or update the public documentation portal. The site
must publish reviewed synthesis, not raw private memory, and it must look like
KnowMe rather than default documentation-framework output.

## Prerequisites

```text
Verify content owner, public/private classification, brand source, site
framework choice, GitHub Pages target, OpenGraph image, search strategy, and
container delivery need.
```

## Discovery and Feynman prompts

```text
Read documentation source tree, content-source manifest, KnowMe UI standard,
Flat 2.0 rules, Docusaurus/Starlight decision notes, Pages workflow, sanitizer,
and OpenGraph assets.
```

```text
Explain how content moves from private/project memory into public docs safely,
how brand tokens implement Flat 2.0, and how Pages/container publication is
proven.
```

## KBD prompts

```text
/kbd-assess branded-docs-portal
Assess content taxonomy, private/public boundary, branding, navigation, search,
Mermaid, Pages URL, container, screenshots, accessibility, and OpenGraph
thumbnail behavior.
```

```text
/kbd-analyze branded-docs-portal
Analyze docs, site config, theme CSS, content source manifest, sanitizer
patterns, GitHub Actions, Pages settings, static assets, and generated routes.
```

```text
/kbd-spec branded-docs-portal
Specify canonical source ownership, sanitized content classes, KnowMe branding,
no-border Flat 2.0 themes, Mermaid/search, Pages workflow, docs container,
OpenGraph tags, accessibility gates, and screenshots.
```

```text
/kbd-plan branded-docs-portal
Plan content classification first, then source migration, theme tokens,
navigation/search/Mermaid, sanitizer, OpenGraph asset, Pages/container build,
visual/a11y checks, critic, and retention.
```

## Implementation and verification

```text
Edit canonical docs and site source only. Do not publish raw memory or private
paths. Use supported theme variables before swizzling. Keep all default branding
removed.
```

```text
Run sanitize, schema checks, production build, local serve or Pages URL check,
representative route checks, OpenGraph metadata inspection, and light/dark
visual review.
```

## Stop evidence

Stop for private raw content, default branding, missing OpenGraph image, broken
build, unverified Pages publication, inaccessible navigation, or unsupported
framework claims.
