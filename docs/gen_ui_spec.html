02 Design Philosophy

The Right Language for Each Layer

Four interlocking principles drive every architectural decision. Each concern lives at the layer where it has the best combination of performance and information.

Language Stratification

Dart owns pixels and reactive state. Rust owns bytes and threads. The FFI boundary is essentially free — flutter_rust_bridge generates zero-copy marshalling with no heap duplication where avoidable.

Single Runtime Principle

One Tokio runtime per process, globally owned. All async tasks share one work-stealing scheduler. The blocking pool (max 8 threads) handles GGUF loading and inference forward passes — never mixing them with SSE event loops.

Sealed Content Block as Protocol

The ContentBlock sealed union is not just a data type — it is the protocol between the agent runtime and the UI. Semantic annotation happens once, at the stream boundary, in the layer with full context.

Locality-First Agent Loop

The PMPO loop runs entirely in Rust, adjacent to both the network and state. Tool dispatch, memory hydration, and result caching happen before Dart sees a single event. The max_turns budget enforces loop termination structurally.

Core Invariant

Push semantic annotation as early as possible; let downstream code stay dumb and declarative. The compiler enforces exhaustiveness at every layer boundary.

05 Content Blocks

Content Block System

The ContentBlock sealed union is the central data primitive of the entire application. It is simultaneously the Dart model, the widget dispatch table, and the UI-layer projection of the A2UI protocol schema. Eleven variants, each mapped exhaustively to a dedicated widget by the compiler.

Adding a new ContentBlock variant produces compile errors at: MessageBubble._buildBlock() and A2uiContentDriver._handle(). The compiler guides you to every location that needs updating — you cannot forget a case.

06 Protocols

A2UI Protocol

A2UI (Agent-to-UI — gen_ui’s internal protocol; not Google’s A2UI standard at a2ui.org — see docs/corrections-2026-07-16.md) is the semantic event envelope that sits above the raw Anthropic SSE stream. Where raw SSE delivers content_block_delta bytes, A2UI delivers structured, named events with typed payloads that map 1:1 to renderable UI blocks.

Figure 4 — A2UI Pipeline: SSE → StreamEvent → A2uiAdapter → FFI → Dart → ContentBlock
07 Protocols

AG-UI Protocol — Bidirectional

AG-UI (CopilotKit open standard) extends A2UI with bidirectionality. Where A2UI is agent→UI only, AG-UI supports frontend→agent events: interrupts, confirmations, state requests. The AguiClient Riverpod notifier is the Flutter-side owner of agent state, pending confirmations, and generative UI render slots.

Figure 5 — AG-UI Bidirectional Protocol: Agent State · Confirmations · Generative UI Slots

Human-in-the-Loop Pattern

1
Agent signals confirmation needed
Before executing a destructive action, the agent emits A2uiConfirmationRequest with a prompt and option list. The Rust runtime parks a oneshot::Receiver<String> in a DashMap keyed by confirmation_id.
2
Flutter renders confirmation dialog
AguiConfirmationOverlay listens to the AguiClient provider. When a new confirmation arrives, showDialog() is called with the options. No polling, no timers — pure reactive.
3
User responds
AguiClient.confirm(id, option) calls ffi.aguiSendClientEvent(), which routes to the Rust AguiClientEventRouter, resolves the oneshot, and resumes the PMPO loop.
4
Agent loop resumes
The confirmation is appended to the conversation as a tool result. The next agent turn has full context of what was approved. No blocking thread was held during the wait.
08 Protocols

Dual Protocol Pipeline

The ProtocolPipeline drives a single mpsc::Receiver<StreamEvent> through both adapters sequentially. A2UI events are produced first (no re-parsing), then passed directly to AguiAdapter::translate(). Two broadcast channels fan out to any number of subscribers with zero duplication.

09 Infrastructure

Local Inference Engine

On-device LLM inference using candle with GGUF quantized models downloaded from HuggingFace Hub. Metal-accelerated on Apple Silicon; rayon BLAS multi-core on Android. The inference loop runs entirely on the Tokio blocking thread pool — never touching the SSE event loop.

Sampling Pipeline

Each token passes through: repeat penalty (penalizes recently generated tokens) → temperature scaling → softmax normalization → top-k filtering → top-p (nucleus) filtering → weighted random sample. All sampling runs synchronously on the blocking thread; tokens stream to Dart via bounded mpsc::channel(256).

11 Infrastructure

MCP Client Architecture

The MCP registry manages N servers simultaneously. Tool names are namespaced as server_name::tool_name to avoid collisions across servers. The agent runtime routes tool calls through the registry transparently — the PMPO loop has no knowledge of which server handled a call.

TransportProtocolConnection PatternStatus
SSE Transport GET {url}/sse → POST {endpoint} Persistent SSE channel; JSON-RPC over POST. Responses return via SSE channel using request ID correlation. Implemented
Stdio Transport subprocess stdin/stdout Spawn process, write JSON-RPC to stdin, read from stdout. Used for local MCP servers like filesystem, git. Stub — wire tokio::process

Tool Name Routing

// Qualified call — explicit server
mcp_reg.call_tool("tavily::web_search", args).await

// Unqualified — scans all servers for first match
mcp_reg.call_tool("web_search", args).await

// Local Rust tool — no network hop, sub-microsecond
agent.register_tool("memory_recall", "...", schema, |args| {
    Ok(format!("recalled: {:?}", args.get("key")))
});

15 Delivery

Extension Guide

Adding a New Content Block Type

The compiler enforces every step. You cannot skip step 4 or 6 — they produce exhaustive switch errors.

1
Add variant to ContentBlock
In gen_ui/lib/features/chat/models/message.dart, add a new const factory ContentBlock.myBlock({...}) = MyBlock; variant.
2
Regenerate freezed
dart run build_runner build — generates the .freezed.dart and .g.dart files.
3
Create the widget
Create widgets/blocks/my_block.dart. Follow the token system — use T.blockDecoration(bg:..., accent:...) for consistent visual language.
4
Add case to MessageBubble._buildBlock()
Compiler enforced. The exhaustive switch in message_bubble.dart fails to compile until this case is added.
5
Add StreamEvent variant (Rust)
In rust/gen_ui_core/src/streaming.rs, add a new arm to the StreamEvent enum. Add ingestion logic to A2uiAdapter::ingest().
6
Add A2uiEvent Dart class + driver case
Add the sealed subclass in bridge/a2ui/a2ui_event.dart. Add the mapping case in A2uiContentDriver._handle(). Compiler enforced.
7
Run frb codegen
If the Rust FFI surface changed, re-run flutter_rust_bridge_codegen generate to regenerate lib/bridge/generated_api.dart.

Connecting the Real FFI Bridge

Three changes activate the Rust backend for a production build.

// 1. In lib/bridge/rust_bridge_provider.dart
import 'generated_api.dart' as ffi;  // uncomment
// replace stub bodies with ffi.* calls

// 2. In lib/features/chat/providers/chat_provider.dart
// Replace _simulateA2UI() with:
final stream = streamAgentA2ui(AgentRunRequest(
  history: _buildFlatHistory(),
  userMessage: text,
  config: const AgentConfig(),
));
_driver = A2uiContentDriver(
  messageId: assistantId,
  onBlock: streamBlock,
  onFinalize: finalizeMessage,
)..connect(stream);

// 3. In lib/main.dart
await initRustBridge(dataDir: appDir.path);
// Local chat is ready without credentials. Optional cloud providers are
// configured through the Rust provider-admin API and platform secure storage.
Integration Checklist

Run build scripts → run frb codegen → uncomment imports → replace simulation → call initRustBridge() before runApp(). All other app code is unchanged — the simulation and production paths share the same ContentBlock model and widget tree.
