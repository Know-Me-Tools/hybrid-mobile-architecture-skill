# C-005 sync-engine — DONE (claude/opus-4.8, in-session)

Implements the `gen_ui_db::sync` module: a Rust Electric shape consumer + DIY write
queue behind the frozen `gen_ui_types::sync::SyncTransport` seam (PES/PSyncV1-compatible).

## Files modified
- `scripts/scaffold-rust-core.sh` — replaced the `emit_l2_stub gen_ui_db` one-liner with
  a dedicated `emit_gen_ui_db()` emitter. C-003 (relational) and C-004 (graph) still own
  their sibling submodules; this change lands only `sync/` + the crate-level deps it needs
  (native-only `reqwest`/`tokio`/`tokio-stream`; wasm32 compiles the native engine out).

## Files the scaffold now emits (crates/gen_ui_db/src/)
- `lib.rs` — declares `pub mod sync`; documents per-lane module ownership.
- `sync/mod.rs` — engine assembly, cfg-gated native vs wasm, re-exports the frozen seam.
- `sync/config.rs` — `SyncConfig` / `ShapeSpec` (builder-style; tenant `where` seam).
- `sync/seam.rs` — `LocalStore` (read-path sink, C-003), `WriteSink` (write-path, C-006
  forge Quarry), `RowChange`/`RowOp`, `PendingWrite`, `WriteOutcome`. Keeps sync free of
  any dependency on the relational or forge crates; PES can reuse the same seams.
- `sync/status.rs` — `SyncStatusHandle` (native `tokio::watch`, last-value-wins for the
  UI chip; wasm `Rc<RefCell>` stub) + atomic pending-writes counter.
- `sync/shapes.rs` — Electric `/v1/shape` HTTP consumer: `(handle, offset)` tracking,
  `offset=-1` initial sync, `live=true` long-poll, `up-to-date`/`must-refetch` control
  frames, HTTP 409 shape-rotation → truncate + refetch, JSON row-message decode.
- `sync/write_queue.rs` — DIY queue: idempotent keys, capped exponential backoff
  (200ms→30s), poison quarantine after `max_write_attempts`, `tokio::Mutex` (guard held
  across sink `.await`, so not parking_lot).
- `sync/engine.rs` — `SyncEngine: SyncTransport`. `start()` spawns one shape consumer per
  shape + a write-drain loop on the global runtime (`gen_ui_runtime::spawn`, no second
  Tokio). `enqueue_write()` parses the mutation, derives a stable FNV-1a idempotency key.
- `sync/README.md` — the web path: `@electric-sql/pglite-sync` JS configuration (Rust
  engine is compiled out on wasm32), with the shared write-path/idempotency contract.

## Deviations / notes
1. wasm `SyncStatusHandle` cannot `#[derive(Default)]` — `SyncStatus` is a frozen seam
   with no `Default` impl. Wrote an explicit `new()`/`Default` constructing
   `SyncStatus::Offline` instead (did NOT touch the frozen gen_ui_types seam, per scope).
2. Write-queue durability (survive restart) is delegated to the `LocalStore`-backed
   action-log table C-003 owns; the in-memory `VecDeque` here is the replay engine and the
   seam is documented — persistence wires in without changing replay logic.
3. Tenant-scoped/authenticated Electric URL + shape `where` RLS filter are left as the
   `ShapeSpec.with_where` + `SyncConfig.electric_url` seams for C-006 to populate.
No blockers. gen_ui_types trait seams untouched (frozen). scripts/ change is additive
(the gen_ui_db stub was a placeholder; other lanes fill their own submodules).
