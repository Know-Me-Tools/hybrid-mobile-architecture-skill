## KEY FINDINGS (detail in references/rust/wasm-targets.md)
1. **MSRV ‚â• 1.96 required for SurrealDB.** `surrealdb 3.2.1 ‚Üí fastnum 0.7.5`
   requires rustc ‚â• 1.94; the workspace `rust-toolchain.toml` pins 1.93 and FAILS
   to build it. ‚Üí **C-004 must bump the toolchain before wiring surrealdb**;
   **C-008 must fix the "Rust 1.80+" tables** in CLAUDE.md/AGENTS.md ‚Üí 1.96+.
2. **SurrealDB `kv-indxdb` is genuinely wasm-safe** ‚Äî analysis ¬ß1.3's claim holds
   under the compiler. Engine URL `indxdb://<name>` via `engine::any`.
3. **3.x API gotchas for C-004:** `Query::take::<R>()` bounds `R: SurrealValue`
   (not serde::Deserialize; `serde_json::Value` rejected) ‚Äî use
   `surrealdb::types::Value` or derive `SurrealValue`. `Value` lives at
   `surrealdb::types::Value`, not `surrealdb::Value`.
4. **reqwest/reqwest-eventsource have no wasm backend** ‚Äî web transport = `web_sys`
   `fetch` + `EventSource` (the wasm impl of the existing Transport/streaming
   seams; still inside the Rust core, no UI-layer networking).
5. **PGlite is JS-owned** ‚Äî only reachable via wasm-bindgen `extern` over a JS
   shim (C-007 ships `js/pglite_shim.js`). No native-Rust sqlx/tokio-postgres
   path on web. Confirms the per-platform matrix (web=PGlite-via-JS,
   desktop=pglite-oxide, mobile=SQLite).
