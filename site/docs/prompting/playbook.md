# Learn, build, prove, retain

The Prometheus control loop is:

```text
Feynman learn → KBD assess → research → decision-complete plan
→ bounded implementation → public-boundary verification
→ adversarial critic → Karpathy retention → next waypoint
```

Every prompt names the outcome, sources of truth, authority, skill sequence,
artifacts, observable verification, limits, stop condition, and retention point.
Use a balanced model for normal implementation, a stronger architecture model for
cross-repository decisions, and an independent model family as critic. Model routing
comes from the dated registry—not copied context windows or vendor claims.

The full guide includes ten staged recipe packs: full hybrid, Flutter-only, Tauri
local agent, automation, SaaS, local agent client, ideation studio, native Rust
agent, multi-cloud deployment, and branded documentation.
